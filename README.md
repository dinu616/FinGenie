# Multi-Agent Customer Analysis Workflow

A modular Python Data Analysis pipeline using a Supervisor/Graph pattern to analyze customer data and recommend credit cards.

## Features
- **Multi-Agent Architecture**: Separate modules for different data domains.
- **Production-Ready Graph**: Uses `langgraph` for robust state management.
- **Python 3.10 Compatible**: Fully upgraded environment.
- **Mock LLM Support**: Runs without AWS credentials for testing/dev (can switch to real Bedrock via utils).

## Setup

1.  **Create Virtual Environment (Python 3.10)**:
    ```bash
    py -3.10 -m venv .venv
    ```

2.  **Install Dependencies**:
    ```bash
    .venv\Scripts\pip install -r requirements.txt
    ```

3.  **Generate Mock Data**:
    ```bash
    .venv\Scripts\python generate_data.py
    ```

4.  **Run the Workflow**:
    ```bash
    .venv\Scripts\python src/graph.py
    ```

5.  **Run Streamlit UI**:
    ```bash
    .venv\Scripts\streamlit run src/app.py
    ```

## Project Structure
- `src/agents/`: Individual agent logic.
- `src/agents/utils.py`: Shared utilities and Mock Core (StateGraph, LLM).
- `src/graph.py`: Main entry point and orchestration.
- `data/`: Input data files (generated by script).
